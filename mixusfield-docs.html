Mixusfield
UI Library

A clean, modular Lua UI framework for Roblox scripts. Build beautiful floating windows, tabs, toggles, sliders, dropdowns, and notifications with minimal code.

Lua 5.1 Roblox Ready 8 Themes v1.0 Stable
01

Introduction

Mixusfield is a Roblox UI library loaded via loadstring + HttpGet. It provides a floating window system with tabs, components, and notifications — all styled, animated, and ready to use.

ℹ️
What makes it different? Mixusfield uses a declarative API — you describe what you want (a toggle, a slider, a dropdown) and the library handles all rendering, state, and callbacks automatically.

All components fire a Callback function when their value changes. The library runs entirely client-side and works alongside any Roblox executor.

02

Installation

Load Mixusfield at the top of any script using loadstring and game:HttpGet:

lua 
local Mixusfield = loadstring(game:HttpGet(
    "https://raw.githubusercontent.com/LumeCraftors01/Mixusfield/refs/heads/main/Mixusfield.lua"
))()
⚠️
HTTP Requests must be enabled Make sure your executor has HTTP requests enabled. The script fetches the library from GitHub every time it runs.
1

Open your executor

Launch your preferred Roblox executor (Synapse, KRNL, Fluxus, etc.) and attach it to Roblox.

2

Paste the loadstring

Copy the loadstring above and paste it at the very top of your script, before any other code.

3

Execute

Run your script. The library loads from the internet and the Mixusfield variable is now ready to use.

03

Quick Start

A minimal working example — creates a window with one tab and a toggle:

lua — full example 
-- 1. Load the library
local Mixusfield = loadstring(game:HttpGet("..."))()

-- 2. Create a window
local Window = Mixusfield:CreateWindow({
    Name     = "My Script",
    Subtitle = "by me",
    Theme    = "Dark",
    ToggleButton = { Enabled = true }
})

-- 3. Create a tab
local Main = Window:CreateTab({
    Name = "Main",
    Icon = { Type = "Lucide", Source = "home" },
})

-- 4. Add components
Main:CreateSection({ Name = "Controls" })

Main:CreateToggle({
    Name    = "Enable Feature",
    Default = false,
    Callback = function(value)
        print("Toggle is now:", value)
    end,
})
04

CreateWindow

Creates the main floating window. Call this once at the top of your script.

lua 
local Window = Mixusfield:CreateWindow({
    Name         = "Window Title",
    Subtitle     = "Subtitle text",
    Theme        = "Dark",
    ToggleButton = { Enabled = true }
})
PropertyTypeRequiredDescription
NamestringrequiredTitle shown at the top of the window
SubtitlestringoptionalSmaller text below the title
ThemestringoptionalUI color theme. Defaults to "Dark"
ToggleButton.EnabledbooloptionalShow a button to hide/show the window

You can change the theme at runtime:

lua
Window:SetTheme("Ocean")  -- change theme live
05

CreateTab

Tabs are the pages inside your window. Each tab holds its own components.

lua
local Tab = Window:CreateTab({
    Name = "Farm",
    Icon = { Type = "Lucide", Source = "activity" },
})
PropertyTypeRequiredDescription
NamestringrequiredTab label shown in the tab bar
Icon.TypestringoptionalIcon system — use "Lucide"
Icon.SourcestringoptionalLucide icon name e.g. "home", "settings", "dollar-sign"
💡
Lucide icon names Browse all available icon names at lucide.dev/icons. Use the lowercase kebab-case name as the Source value.
06

Components

All components are methods on a Tab object. Click each card to expand full details and code.

CreateSection layout ▶

Adds a labeled divider to visually group components inside a tab.

lua
Tab:CreateSection({ Name = "Section Title" })
CreateToggle toggle ▶

A boolean on/off switch. Fires Callback with true or false when clicked.

lua
Tab:CreateToggle({
    Name     = "Enable Feature",
    Default  = false,       -- starting state
    Callback = function(value)
        -- value is true or false
        print(value)
    end,
})
PropertyTypeDescription
NamestringLabel shown next to the toggle
DefaultboolInitial state (true = on)
CallbackfunctionCalled with current value on change
CreateSlider slider ▶

A draggable numeric slider with min, max, default, and increment.

lua
Tab:CreateSlider({
    Name      = "Speed",
    Min       = 0.2,
    Max       = 5,
    Default   = 1,
    Increment = 0.1,
    Suffix    = " sec",     -- shown after the value
    Callback  = function(value)
        print("Slider:", value)
    end,
})
PropertyTypeDescription
Min / MaxnumberMinimum and maximum values
DefaultnumberStarting value
IncrementnumberStep size when dragging
SuffixstringUnit label appended to value display
CallbackfunctionCalled with current number value
CreateButton action ▶

A clickable button that fires Callback once each press.

lua
Tab:CreateButton({
    Name     = "Do Action",
    Callback = function()
        print("Button clicked!")
    end,
})
CreateDropdown select ▶

A dropdown list. Callback fires with the selected string each time the user picks an option.

lua
Tab:CreateDropdown({
    Name    = "Select Item",
    Options = { "Option A", "Option B", "Option C" },
    Callback = function(value)
        print("Selected:", value)
    end,
})

To refresh the dropdown options dynamically:

lua
local dd = Tab:CreateDropdown({...})
dd:Refresh({ "New A", "New B" })  -- replace options
CreateParagraph display ▶

Displays static text with a bold title and body content. Great for pricing info, instructions, or notes.

lua
Tab:CreateParagraph({
    Title   = "Item Prices",
    Content = "Plastic Pan → $500\nMetal Pan → $12K",
})
💡
Use \n inside the Content string to add line breaks.
07

Notifications

Show toast notifications from anywhere in your script using Mixusfield:Notify().

lua
Mixusfield:Notify({
    Title   = "Purchased!",
    Content = "Golden Pan bought for $333K",
    Type    = "success",   -- "success" | "error" | "warning" | "info"
})
Type valueColorWhen to use
success■ GreenPurchases, toggles on, actions completed
error■ RedFailed purchases, missing character, exceptions
warning■ YellowMissing selection, validation issues
info■ BlueGeneral information, state changes

Prospecting Hub — Overview

The Prospecting Hub is a ready-to-use script built on Mixusfield for gold prospecting games. It includes 5 tabs covering automation, teleportation, shopping, selling, and settings.

✅
What the Hub does Automatically pans for gold, teleports your character forward during each pan shake and returns you, buys items from the in-game shop via RemoteFunction, sells your inventory, and lets you customize the UI theme.
TabPurpose
MainToggle Auto Prospect on/off, set speed, manual prospect
TeleportConfigure forward stud distance, hold timer, auto run, test teleport
ShopBuy pans, shovels, potions, and items from all zones
SellSell All button + Auto Sell toggle (every 3 seconds)
SettingsChange UI theme, reset all systems
10

Hub Tab Guide

🏠 Main Tab auto ▶

Controls the core auto-prospecting loop.

1

Enable Auto Prospect

Toggle ON to start the auto loop. The script fires Collect, Pan, and Shake remotes on your equipped pan every N seconds.

2

Prospect Speed slider

Adjust from 0.2–5 seconds between each prospect cycle. Lower = faster farming.

3

Manual Prospect Once

Fire one prospect cycle immediately without enabling auto loop.

🚀 Teleport Tab movement ▶

Controls how Auto Prospect moves your character during each cycle.

1

Forward Distance

Set 1–50 studs. During each auto prospect, the character is moved forward by this many studs (X/Z only — height never changes).

2

Teleport Hold Timer

0–3 seconds to wait at the forward position before firing remotes and returning. Tweak this if the server needs more time to register the position.

3

Auto Run toggle

Continuously moves your character forward (like holding W). Disable when not needed to prevent drifting.

4

Test button

Teleports forward by your set distance, waits the hold timer, then returns you. Use this to verify settings before enabling Auto Prospect.

🛒 Shop Tab buy ▶

Buy any item from the in-game shop. Items are organized into 4 sections.

ℹ️
Each section has a dropdown to pick the item and a Buy button to purchase it. Items are sorted cheapest to most expensive. A pricing paragraph shows prices and zones for reference.
SectionItems
🍳 Pans14 pans — Plastic Pan ($500) through Lifetouched Pan ($80B)
⛏️ Shovels17 shovels — Iron Shovel ($3K) through Lifetouched Shovel ($80B)
🧪 Potions11 potions with effects — Basic Luck ($5K) to Supreme Luck ($500K)
📦 Items23 items — sluice boxes, totems, special items, and more

The buy function uses workspace.Purchasable.[Zone].[ItemName].ShopItem and calls Remotes.Shop.BuyItem:InvokeServer(shopItem, 1).

💰 Sell Tab sell ▶

Sell everything in your backpack instantly or automatically.

1

Sell All Now

Fires Remotes.Shop.SellAll:InvokeServer() once. All items in your backpack are sold immediately.

2

Auto Sell All toggle

Enable to automatically sell everything every 3 seconds. Combine with Auto Prospect for fully automated farming.

⚙️ Settings Tab config ▶

Theme selection and system reset.

🎨
UI Theme dropdown — pick from Dark, Light, Midnight, Ocean, Forest, Sunset, Candy, or Rose. The window updates live via Window:SetTheme().

Reset All Systems turns off Auto Prospect, Auto Sell, Auto Run, resets all sliders to defaults, and clears all dropdown selections.

Loader Script

To run the Prospecting Hub, execute this one-liner in your Roblox executor:

lua — execute this 
loadstring(game:HttpGet("https://raw.githubusercontent.com/megaloton1345-ui/jdjsjja/refs/heads/main/ProspectingHub.lua.txt"))()
⚠️
Executor requirements Your executor must support game:HttpGet, loadstring, and CFrame manipulation. Most modern executors (Synapse X, Fluxus, Arceus X, etc.) support all of these.
ℹ️
Auto updates Because the script is loaded from a URL every time, any updates pushed to the GitHub repository will automatically be applied next time you run the loader.
